üìã Understanding XDM Schema

What is XDM Schema? ü§î

XDM = Experience Data Model
Think of it as a blueprint or template that tells AEP exactly what your data looks like and means. It's like having a universal translator for all your data!

Schema Components:

1. Field Groups (Mixins)

Pre-built sets of fields for common data types:

‚Ä¢ Profile Demographics (name, age, gender) ‚Ä¢ Contact Details (email, phone, address) ‚Ä¢ Commerce Details (purchases, cart data) ‚Ä¢ Web Analytics (page views, clicks)

2. Data Types

Different kinds of information you can store:

  • String: Text data like names, addresses
  • Number: Quantities, prices, ages
  • Boolean: True/false values (subscribed/not subscribed)
  • Date: Timestamps, birthdates
  • Object: Complex data with multiple properties

Creating Your First Schema - Step by Step:

  1. Go to AEP ‚Üí Schemas ‚Üí Create Schema
  2. Choose "XDM Individual Profile" or "XDM ExperienceEvent"
  3. Add Field Groups that match your data needs
  4. Add custom fields if needed
  5. Set Identity fields (email, customer ID)
  6. Enable for Profile if you want unified customer views
  7. Save and activate your schema

üì• Data Ingestion Made Simple

What is Data Ingestion? üöö

Data ingestion is like moving furniture into your new house (AEP). You need to get your data from where it currently lives into Adobe Experience Platform so you can use it.

Two Main Types:

1. Batch Ingestion üì¶

Upload data files all at once - like moving boxes

  • Best for: Historical data, daily reports, large CSV files
  • Formats: CSV, JSON, Parquet
  • How: Upload files through UI or API
  • Speed: Processes every few hours or daily

2. Streaming Ingestion ‚ö°

Data flows in real-time - like a water stream

  • Best for: Website clicks, app events, live transactions
  • Method: API calls, SDK integration
  • Speed: Data available within seconds
  • Use case: Real-time personalization

Step-by-Step: Your First Batch Upload

  1. Prepare your CSV file with proper headers
  2. Go to AEP ‚Üí Sources ‚Üí Add Data
  3. Choose "Upload File" as your source
  4. Map your CSV columns to schema fields
  5. Preview and validate the mapping
  6. Configure the dataset name and description
  7. Start the ingestion process
  8. Monitor progress in the monitoring dashboard
Example CSV Structure: email,firstName,lastName,age,city john@email.com,John,Doe,25,New York jane@email.com,Jane,Smith,30,Los Angeles

üë• Customer Segmentation

What are Segments? üéØ

Segments are groups of customers who share similar characteristics or behaviors. Think of it like organizing your friends into groups - sports fans, foodies, travelers - so you can send them relevant invitations!

Types of Segments:

1. Batch Segments üìä

  • When: Evaluated daily or on schedule
  • Best for: Complex analysis, historical data patterns
  • Example: "Customers who bought 3+ items last month"

2. Streaming Segments ‚ö°

  • When: Updated in real-time as data arrives
  • Best for: Immediate actions, live personalization
  • Example: "Visitors currently on checkout page"

Building Your First Segment:

  1. Go to AEP ‚Üí Segments ‚Üí Create Segment
  2. Choose your evaluation method (Batch/Streaming)
  3. Define your audience using the segment builder
  4. Add attributes (age, location, purchase history)
  5. Set up events (page views, purchases, clicks)
  6. Preview estimated audience size
  7. Save and activate your segment
